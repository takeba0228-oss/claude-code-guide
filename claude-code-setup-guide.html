はじめに＋用語の解説

Claude Code = 日本語で「こういうの作って」と話しかけるだけで、AIがコードを自動で書いてくれるツール。
CMUX = Claude Code を快適に使うための Mac 専用アプリ（無料）。

Claude の有料プラン（Pro / Max）月額サブスク。これに入ると Claude Code が使える
▼
Claude CodeAIアシスタント本体
▼
CMUX（シーマックス）Claude Code を動かす専用アプリ（無料）

「Claude Code のアプリもあるのに、なぜ CMUX？」

よくある疑問

Claude Code にはデスクトップアプリ版もありますが、CMUX を使う理由があります：

Claude Code
デスクトップアプリ		CMUX
画面分割できないできる（複数同時作業）
複数AI同時起動できないできる（チーム機能）
ワークスペース管理なしプロジェクトごとに整理
ショートカット少ない豊富（作業が速い）
料金無料無料

つまり CMUX = Claude Code をもっと便利に使うためのパワーアップツール。無料なので入れない理由がありません。

Claude Code のアプリを持っていても、ターミナルでのインストールは必要です！

Claude Code のデスクトップアプリ（Mac アプリ）を既にインストールしていても、CMUX から使うにはターミナルでのインストール（このステップ6）が別途必要です。

6-1. まず Node.js が入っているか確認

この後 CMUX で claude と打った時に Error: claude not found in PATH と出る原因の多くは Node.js が入っていないことです。必ずこのステップを実行してください。

作ったファイルはどこ？見方と探し方
超重要！初心者が一番つまずくポイント

Claude が「ファイルを作成しました」と言ったけど、「どこにあるの？」「どうやって見るの？」と思いますよね。
ここで全部説明します。

ファイルの保存場所

Claude が作ったファイルは、claude コマンドを実行したフォルダに保存されます。

「どこで claude を起動したか」でファイルの場所が決まる

claude だけで起動→ホーム（~）= Finder の家のアイコン
cd ~/Desktop してから claude→デスクトップに保存される
cd ~/Documents/my-site してから claude→Documents/my-site に保存される

作ったファイルを見る3つの方法

1
方法1：Claude に開いてもらう（いちばん簡単！）
Claude に「ブラウザで開いて」と言うだけ。
あなた:
今作ったページ、ブラウザで見せて
Claude:
Bash open index.html
→ 1 を押すと Safari でページが開く！
2
方法2：Finder で探す
Finder（笑顔のアイコン）を開く →
左メニューの家のアイコン（ホーム）をクリック →
Claude が作ったファイルがここにあります。
家のアイコンが見つからない？

Finder メニューバー →「移動」→「ホーム」（ショートカット：Command+Shift+H）

3
方法3：CMUX のターミナルで確認
ls と打つと、今いるフォルダのファイル一覧が表示されます。
cmux
% ls
Desktop Documents Downloads index.html style.css
↑ index.html と style.css が Claude が作ったファイル！

HTML ファイルをブラウザで見る方法

超簡単！

Finder で .html ファイルをダブルクリックするだけ。Safari か Chrome で開きます。
または、ファイルをブラウザのウィンドウにドラッグ＆ドロップしてもOK。

おすすめ：最初からデスクトップで作業する

ファイルの場所に迷わないよう、デスクトップにフォルダを作ってから始めるのがおすすめ：

cmux — 最初にこれを実行
% mkdir ~/Desktop/my-site ← デスクトップに「my-site」フォルダを作成
% cd ~/Desktop/my-site ← そのフォルダに移動
% claude ← ここで Claude を起動！

こうすると、作ったファイルが全部デスクトップの「my-site」フォルダに入るので、Finder ですぐ見つかります。

「ファイルが何個もできた！」ときの見方

Webサイトを頼むと、Claude は複数のファイルを作ることがあります。慌てなくて大丈夫！

Finder でフォルダを開くとこんな感じ
my-site/
├── index.html ← メインのページ（これをブラウザで開く！）
├── style.css ← デザイン（色、フォント、レイアウト）
├── script.js ← 動き（ボタンクリック、アニメーション）
└── images/ ← 画像フォルダ
ファイル名何これ？開く必要は？
.htmlページの中身これをダブルクリック！
.css見た目のデザイン触らなくてOK（自動で読まれる）
.js動きの指示触らなくてOK（自動で読まれる）
images/画像の入れ物画像を追加したいとき

ポイント：開くのは .html だけ！他のファイルは .html が自動で読み込むので、触らなくて大丈夫です。

次の日の続き方（超重要！）

CMUX を閉じても作ったファイルは消えません！

ただし会話の履歴は消えます（Claude は前回何を話したか忘れます）。
でもファイルは残っているので、翌日もちゃんと続きから作業できます。

翌日の再開手順：

エラー別トラブルシューティング

データ保護・バックアップ
重要
Claude Code で作った会話やファイル、消えたらもう取り戻せません。
PCの故障・誤操作・再インストールに備えて、バックアップを設定しておきましょう。

Claude Code のデータはどこに保存されている？

Claude Code の全データは Mac の中の隠しフォルダに入っています：

~/.claude/
Claude Code のホームフォルダ。全てのデータがここに入っている
会話ログ（.jsonl ファイル）
Claude とのやり取りがすべて記録されている。claude --resume で過去の会話を再開できるのはこのおかげ
メモリ（memory フォルダ）
Claude に「覚えて」と言った内容が保存される場所。会話をまたいで記憶を引き継ぐ
設定ファイル（settings.json）
スキル・フック・権限設定など、カスタマイズした内容
注意
これらは Mac のローカルにしか存在しません。PC が壊れたら全部消えます。Anthropic のサーバーには保存されていません。
